代码映射数仓建模汇总

从源绑定 DWD 明细层出发，经过双源校验，生成去来源信息的 DWS CFI 统一代码映射表。

项目目录：/home/dsou/code_mapping_jydb_datayes_ab/

一、建模链路

DWD 源绑定明细层聚源：dwd_gildata_range_year
通联：dwd_datayes_range_year
保留源内 ID，便于追溯。
→
双源校验层统一字段后按拉链键比对，差异保留在 validation 结果中。
→
DWS CFI 统一层dws_cfi_range_year
去来源、去源内 ID，仅保留下游消费字段。

二、源表构造简述

聚源基于 SecuMain + DZ_CodeChange + DZ_ListStatus 构造代码拉链。

通联基于 md_security + md_sec_chg + md_trade_cal 构造同口径拉链，并将代码变更结束日转为右开区间边界。

三、区间语义

所有拉链表统一采用：

[strart_date, stop_date)

  • strart_date 当天有效
  • stop_date 当天不再有效
  • stop_date 为空表示当前开口

四、子表拆分

A股 / 沪深：a_share_secu_mapping_z A股 / 北交：a_share_bjse_secu_mapping_z B股 / 沪深：b_share_secu_mapping_z

类别和市场由表名表达，表内不再保留 exchange 字段。

五、DWD 源绑定明细层

来源测试库字段用途
聚源 Gildata dfs://user_dsou_dwd_gildata_range_year cfi_prod_time, instrument, listed_date, delist_date, secucode, innercode, companycode, strart_date, stop_date 保留聚源交易代码、证券级 ID 和公司级 ID。
通联 Datayes dfs://user_dsou_dwd_datayes_range_year cfi_prod_time, instrument, listed_date, delist_date, ticker_symbol, security_id, party_id, strart_date, stop_date 保留通联交易代码、证券级 ID 和主体级 ID。

六、DWS CFI 统一消费层

目标测试库：dfs://user_dsou_dws_cfi_range_year

统一字段：

cfi_prod_time, instrument, listed_date, delist_date, ticker, strart_date, stop_date

映射关系：

  • 聚源：secucode → ticker，去掉 innercode/companycode
  • 通联：ticker_symbol → ticker，去掉 security_id/party_id

当前 DWS CFI 默认使用通联明细层作为主源，聚源明细层用于校验和追溯。

七、校验逻辑

  • 字段完整性检查
  • strart_date 非空
  • stop_date >= strart_date 或为空
  • 拉链行不重复
  • DDB 行数 = CSV 行数

八、双源校验

两源统一后按以下键比对：

instrument + ticker + strart_date + stop_date

差异分为：一致聚源独有通联独有

当前截面完全一致

历史拉链有少量差异，保留在 validation 结果中，不进入 DWS 字段。
